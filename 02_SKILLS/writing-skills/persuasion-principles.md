# 스킬 설계를 위한 설득 원칙

## 개요

LLM은 인간과 같은 설득 원리에 반응한다. 이 심리를 이해하면 압박 상황에서도 중요한 실천이 지켜지도록, 더 효과적인 스킬을 설계하는 데 도움이 된다. 목적은 조작이 아니라 핵심 실천을 안정적으로 따르게 만드는 것이다.

**연구 근거:** Meincke et al. (2025)는 AI 대화 28,000건(N=28,000)을 대상으로 7가지 설득 원칙을 검증했다. 설득 기법을 사용했을 때 준수율은 33%에서 72%로 두 배 이상 증가했다(p < .001).

## 일곱 가지 원칙

### 1. 권위(Authority)
**의미:** 전문성, 자격, 공식 출처에 대한 신뢰와 복종.

**스킬에서 작동하는 방식:**
- 명령형 언어 사용: "반드시 해야 한다", "절대 하지 마라", "항상"
- 협상의 여지가 없는 프레이밍: "예외는 없다"
- 의사결정 피로와 자기합리화를 줄여 준다

**언제 사용할까:**
- 규율을 강제하는 스킬(TDD, 검증 요구사항 등)
- 안전이 중요한 실천
- 이미 확립된 모범 사례

**예시:**
```markdown
테스트보다 먼저 코드를 썼다고? 지워라. 다시 시작하라. 예외는 없다.
권장하지 않음: 가능하다면 테스트를 먼저 작성하는 것을 고려하라.
```

### 2. 헌신(Commitment)
**의미:** 이전 행동, 발언, 공개적 선언과 일관되게 행동하려는 경향.

**스킬에서 작동하는 방식:**
- 선언을 요구한다: "스킬 사용을 선언하라"
- 명시적 선택을 강제한다: "A, B, C 중 하나를 선택하라"
- 추적 수단을 사용한다: 체크리스트용 `todo` 도구

**언제 사용할까:**
- 스킬이 실제로 지켜지도록 보장할 때
- 여러 단계로 이루어진 프로세스
- 책임성을 높이는 장치가 필요할 때

**예시:**
```markdown
스킬을 찾았으면 반드시 이렇게 선언해야 한다: "[Skill Name]을 사용하겠습니다"
NOT: 어떤 스킬을 사용하는지 파트너에게 알려주는 것도 고려해 보라.
```

### 3. 희소성(Scarcity)
**의미:** 시간 제한이나 제한된 기회에서 오는 긴박감.

**스킬에서 작동하는 방식:**
- 시간 제한이 있는 요구사항: "계속 진행하기 전에"
- 순차적 의존성: "X 직후 즉시"
- 미루는 행동을 막는다

**언제 사용할까:**
- 즉시 검증이 필요한 요구사항
- 시간 민감한 워크플로우
- "나중에 하지 뭐"를 방지해야 할 때

**예시:**
```markdown
작업을 완료한 뒤에는 계속 진행하기 전에 즉시 코드 리뷰를 요청하라.
권장하지 않음: 편할 때 코드를 검토해도 된다.
```

### 4. 사회적 증거(Social Proof)
**의미:** 다른 사람들이 하는 방식, 혹은 일반적으로 여겨지는 기준에 따르려는 경향.

**스킬에서 작동하는 방식:**
- 보편적 패턴을 강조한다: "매번", "항상"
- 실패 형태를 명시한다: "Y 없는 X = 실패"
- 규범을 세운다

**언제 사용할까:**
- 보편적으로 적용되는 실천을 문서화할 때
- 흔한 실패를 경고할 때
- 기준과 표준을 강화할 때

**예시:**
```markdown
`todo` 도구로 추적하지 않는 체크리스트 = 단계가 빠진다. 매번 그렇다.
NOT: 어떤 사람들은 체크리스트에 `todo` 도구가 도움이 된다고 느낀다.
```

### 5. 일체감(Unity)
**의미:** 공유된 정체성, "우리"라는 감각, 같은 집단에 속해 있다는 느낌.

**스킬에서 작동하는 방식:**
- 협업적 언어 사용: "우리 코드베이스", "우리는 함께 일하는 동료다"
- 공동 목표를 강조한다: "우리 둘 다 품질을 원한다"

**언제 사용할까:**
- 협업 워크플로우
- 팀 문화 형성
- 위계적이지 않은 실천

**예시:**
```markdown
우리는 함께 일하는 동료다. 당신의 솔직한 기술적 판단이 필요하다.
NOT: 내가 틀렸다면 아마 말해주는 편이 좋을 것이다.
```

### 6. 호혜성(Reciprocity)
**의미:** 받은 이익을 돌려줘야 한다고 느끼는 경향.

**작동 방식:**
- 신중하게만 사용해야 한다. 쉽게 조작적으로 느껴질 수 있다
- 스킬에서는 대체로 필요하지 않다

**언제 피해야 할까:**
- 거의 항상(다른 원칙이 더 효과적이다)

### 7. 호감(Liking)
**의미:** 호감을 느끼는 대상과 더 잘 협력하려는 경향.

**작동 방식:**
- **준수 유도에는 사용하지 말 것**
- 솔직한 피드백 문화와 충돌한다
- 아첨과 영합을 만든다

**언제 피해야 할까:**
- 규율 강제 목적에서는 항상

## 스킬 유형별 원칙 조합

| 스킬 유형 | 사용할 것 | 피할 것 |
|------------|-----|-------|
| 규율 강제형 | 권위 + 헌신 + 사회적 증거 | 호감, 호혜성 |
| 가이드/기법형 | 적당한 권위 + 일체감 | 과도한 권위 |
| 협업형 | 일체감 + 헌신 | 권위, 호감 |
| 참조형 | 명확성만 | 모든 설득 기법 |

## 왜 효과가 있는가: 심리적 배경

**경계가 분명한 규칙은 자기합리화를 줄인다:**
- "반드시 해야 한다"는 의사결정 피로를 없앤다
- 절대적 표현은 "이건 예외인가?" 같은 질문을 없앤다
- 반합리화 문구를 명시하면 구체적인 빠져나갈 구멍을 막을 수 있다

**실행 의도는 행동을 자동화한다:**
- 명확한 트리거 + 요구된 행동 = 자동 실행
- "X일 때 Y를 하라"는 "대체로 Y를 하라"보다 효과적이다
- 준수에 필요한 인지 부하를 줄인다

**LLM은 준인간적(parahuman)이다:**
- 이런 패턴이 담긴 인간 텍스트로 학습되었다
- 학습 데이터에서 권위적 언어는 종종 준수보다 앞서 등장한다
- 헌신 시퀀스(선언 -> 행동)는 자주 모델링되어 있다
- 사회적 증거 패턴(모두가 X를 한다)은 규범을 형성한다

## 윤리적 사용

**정당한 사용:**
- 중요한 실천이 지켜지도록 보장할 때
- 효과적인 문서를 만들 때
- 예측 가능한 실패를 막을 때

**부당한 사용:**
- 개인적 이익을 위한 조작
- 거짓된 긴급성 만들기
- 죄책감을 이용한 준수 강요

**판단 기준:** 사용자가 이를 완전히 이해했을 때에도, 이 기법이 사용자의 진짜 이익에 부합하는가?

## 연구 인용

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 설득의 7가지 원칙
- 영향력 연구의 경험적 기반

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 28,000건의 LLM 대화에서 7가지 원칙을 검증함
- 설득 기법 사용 시 준수율이 33% -> 72%로 증가함
- Authority, commitment, scarcity가 가장 효과적이었음
- LLM 행동의 준인간적(parahuman) 모델을 뒷받침함

## 빠른 참고

스킬을 설계할 때는 다음을 자문하라:

1. **이 스킬은 어떤 유형인가?** (규율 vs. 가이드 vs. 참조)
2. **내가 바꾸려는 행동은 무엇인가?**
3. **어떤 원칙을 적용할 수 있는가?** (규율에는 보통 authority + commitment)
4. **너무 많이 섞고 있지는 않은가?** (일곱 가지를 다 쓰지 말 것)
5. **윤리적인가?** (사용자의 진짜 이익에 부합하는가?)
