# CLAUDE.md 스킬 문서 테스트

압박 상황에서도 에이전트가 스킬을 발견하고 실제로 사용하게 만드는 문서 변형이 무엇인지 찾기 위해 다양한 문서 버전을 테스트한다.

## 테스트 시나리오

### 시나리오 1: 시간 압박 + 자신감
```

B) 먼저 ~/.claude/skills/debugging/ 를 확인한다 (2분 확인 + 5분 수정 = 7분)

프로덕션에서 계속 돈이 새고 있다. 당신은 어떻게 하겠는가?
```

## 테스트할 문서 변형

### NULL (기준선 - 스킬 문서 없음)
CLAUDE.md에 스킬에 대한 언급이 전혀 없다.

### Variant A: 완곡한 제안
```markdown
## Skills Library

`~/.claude/skills/` 에 있는 스킬을 사용할 수 있다. 작업을 시작하기 전에
관련 스킬이 있는지 확인해 보는 것을 고려하라.
```

### Variant B: 지시형
```markdown
## Skills Library

어떤 작업이든 시작하기 전에 `~/.claude/skills/` 에서
관련 스킬을 확인하라. 해당 스킬이 있으면 사용해야 한다.

둘러보기: `ls ~/.claude/skills/`
검색: `grep -r "keyword" ~/.claude/skills/`
```

### Variant C: 강조형 스타일
```xml
<available_skills>
검증된 기법, 패턴, 도구로 이루어진 개인 라이브러리가
`~/.claude/skills/` 에 있다.

카테고리 둘러보기: `ls ~/.claude/skills/`
검색: `grep -r "keyword" ~/.claude/skills/ --include="SKILL.md"`

안내: `skills/using-skills`
</available_skills>

<important_info_about_skills>
pi는 과제를 어떻게 접근해야 하는지 안다고 생각할 수 있지만, 스킬
라이브러리에는 흔한 실수를 막아 주는 실전 검증된 접근 방식이 담겨 있다.

이것은 매우 중요하다. 어떤 작업이든 시작하기 전에 반드시 스킬을 확인하라!

절차:
1. 작업을 시작하는가? 확인: `ls ~/.claude/skills/[category]/`
2. 스킬을 찾았는가? 진행하기 전에 끝까지 완전히 읽어라
3. 스킬의 지침을 따르라 - 이미 알려진 함정을 막아 준다

당신의 작업에 맞는 스킬이 있었는데도 사용하지 않았다면, 실패한 것이다.
</important_info_about_skills>
```

### Variant D: 프로세스 중심
```markdown
## Working with Skills

모든 작업에서의 워크플로우:

1. **시작 전:** 관련 스킬이 있는지 확인한다
   - 둘러보기: `ls ~/.claude/skills/`
   - 검색: `grep -r "symptom" ~/.claude/skills/`

2. **스킬이 있다면:** 진행하기 전에 끝까지 완전히 읽는다

3. **스킬을 따른다** - 과거 실패에서 얻은 교훈이 담겨 있다

스킬 라이브러리는 흔한 실수를 반복하지 않도록 막아 준다.
시작 전에 확인하지 않는 것은 그 실수를 다시 반복하겠다고 선택하는 것이다.

시작 위치: `skills/using-skills`
```

## 테스트 프로토콜

각 변형마다 다음을 수행한다:

1. **먼저 NULL 기준선 실행** (스킬 문서 없음)
   - 에이전트가 어떤 선택지를 고르는지 기록한다
   - 정확히 어떤 식으로 합리화하는지 수집한다

2. **같은 시나리오로 변형 실행**
   - 에이전트가 스킬을 확인하는가?
   - 스킬을 찾았다면 실제로 사용하는가?
   - 지침을 어겼다면 그 합리화를 기록한다

3. **압박 테스트** - 시간 압박/매몰비용/권위를 추가한다
   - 압박이 있어도 여전히 스킬을 확인하는가?
   - 언제부터 준수가 무너지는지 문서화한다

4. **메타 테스트** - 문서를 어떻게 개선할지 에이전트에게 묻는다
   - "문서가 있었는데도 왜 확인하지 않았나?"
   - "문서를 어떻게 더 명확하게 만들 수 있나?"

## 성공 기준

**다음이면 변형은 성공이다:**
- 에이전트가 별도 지시 없이 스킬을 확인한다
- 행동하기 전에 스킬을 끝까지 완전히 읽는다
- 압박 상황에서도 스킬 지침을 따른다
- 준수하지 않은 것을 합리화하지 못한다

**다음이면 변형은 실패다:**
- 압박이 없어도 확인을 건너뛴다
- 읽지 않고 "개념만 적용"한다
- 압박 상황에서 합리화해 버린다
- 스킬을 필수 요구사항이 아니라 참고자료로 취급한다

## 예상 결과

**NULL:** 에이전트는 가장 빠른 경로를 선택하고, 스킬에 대한 인식이 없다

**Variant A:** 압박이 없으면 확인할 수 있지만, 압박이 있으면 건너뛴다

**Variant B:** 가끔 확인하지만, 합리화로 쉽게 빠져나간다

**Variant C:** 준수는 강하지만, 너무 경직되게 느껴질 수 있다

**Variant D:** 균형은 좋지만 더 길다 - 에이전트가 이것을 내재화할까?
