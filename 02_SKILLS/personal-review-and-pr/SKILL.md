# personal-review-and-pr

변경사항 리뷰 → 코드 간소화 → 재리뷰 & 커밋 → PR 생성을 **중간 승인 없이** 끝까지 자동으로 진행하는 워크플로우.

## 사전 조건

- git 워크스페이스에서 실행
- 현재 브랜치에 커밋되지 않은 변경사항 또는 푸시되지 않은 커밋이 있어야 함
- 인자 `$ARGUMENTS`가 있으면 PR target branch로 사용

## 워크플로우

아래 4단계를 **중단 없이** 순서대로 실행해. 각 단계가 끝나면 결과를 한 줄로만 요약하고 곧바로 다음 단계로 진행해. 사용자에게 확인이나 승인을 요청하지 마.

### 1단계: 1차 리뷰 & 커밋 단위 정리

`code-review-and-quality` 스킬을 적용해 변경사항을 리뷰해.
- 자동으로 수정 가능한 이상이 발견되면 직접 고친 뒤 다시 점검해.
- 리뷰가 끝나면 `commit` 스킬을 적용해 커밋 단위(스테이징 그룹)를 정리만 해두고 **아직 커밋은 실행하지 마**.
- 결과를 한 줄로 요약하고 바로 다음 단계로.

### 2단계: 코드 간소화 점검

`code-simplification` 스킬을 적용해.
- 개선 사항이 발견되면 적용해.
- 개선이 없으면 그대로 다음 단계로 진행해.

### 3단계: 2차 리뷰 & 커밋 확정

`code-review-and-quality` 스킬을 다시 적용해.
- 2단계에서 변경이 있었다면 커밋 단위를 재정리해.
- 최종 커밋 단위가 확정되면 **사용자 승인을 받지 않고 바로 `commit` 스킬로 커밋을 실행**해.
- 커밋 결과(해시/메시지)는 한 줄로만 요약해.

### 4단계: PR 생성

`create-pr` 스킬을 적용해 PR을 생성해.
- `$ARGUMENTS`가 있으면 target branch로 전달해.
- PR 생성도 별도 승인 없이 바로 진행해.

## 규칙

- 각 하위 스킬의 규칙은 그대로 따르되, **사용자 승인/확인을 요구하는 부분은 무시**하고 자동으로 진행해.
- 1~2단계에서 수정이 발생하면 3단계에서 반드시 재점검해.
- 단, 다음 경우에는 진행을 멈추고 사용자에게 보고해:
  - 리뷰에서 자동으로 수정할 수 없는 심각한 이슈 발견 (보안 결함, 동작 변경 의심 등)
  - 테스트/빌드/타입 체크 실패 + 자동 수정 불가
  - 명령/스킬 실행 자체가 실패한 경우 (예: `gh` 미인증, push 실패)
- 4단계가 끝난 뒤 1~4단계 결과와 PR 링크를 묶어 **단 한 번만** 최종 보고해.

## 출력 형식 (최종 보고)

```
✅ 1차 리뷰: <요약>
✅ 간소화: <요약>
✅ 2차 리뷰 & 커밋: <커밋 N개> — <대표 메시지>
✅ PR: <PR URL>
```
